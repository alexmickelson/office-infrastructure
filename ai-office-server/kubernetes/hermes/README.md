# Hermes Agent

## First-time setup

Run the setup wizard once to create the `.env` and `config.yaml` on the host node:

```bash
docker run -it --rm \
  -v /data/hermes:/opt/data \
  144.17.92.12:5000/hermes-agent:latest setup
```

This writes `/data/hermes/.env` and `/data/hermes/config.yaml`. Run this on the node
that will host the pod (hostPath volume is not portable across nodes).

## .env file

After setup, `/data/hermes/.env` should contain at minimum:

```env
# LLM provider — at least one required
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

# API_SERVER_ENABLED and API_SERVER_HOST are set as Deployment env vars.
# API_SERVER_KEY is auto-generated by the init container on first run if not present.

# Optional chat platform tokens
# TELEGRAM_BOT_TOKEN=...
# DISCORD_BOT_TOKEN=...
```

## Deploy

```bash
kubectl apply -f application.yml
```

## Secrets

API keys are stored in `/data/hermes/.env` on the host. To manage them as a
Kubernetes Secret instead, create one manually and use `kubectl` or Sealed Secrets —
do not commit keys to this repo.
